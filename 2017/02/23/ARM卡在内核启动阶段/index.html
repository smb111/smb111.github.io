齐鲁工大--Smartbuilding后期维护

发表于 2017-02-23   |   分类于 项目

ARM板启动的时候卡在Uncompressing Linux……………. done, booting the kernel.

1、machine type 不匹配

该部分审查代码后确定machine type是匹配的，排除

2、串口驱动没有编译入内核

该部分审查.config文件后发现串口也配置编入了内核，排除。

3、内核启动参数设置错误

以上时出现该问题的原因，因为使用的是友善之臂的superboot，并不是uboot，不能进入uboot的命令行，无法查看命令行启动参数。但是从出错信息来看这是很明显的内核问题， —要么是superboot的启动参数出错，要么是内核文件出错。使用miniTools重新刷入内核和启动参数，OK！

如果使用uboot便可以进入uboot的命令行查看 启动参数，但是并没有用 uboot，用别人的东西就要被捆绑使用别人的软件
