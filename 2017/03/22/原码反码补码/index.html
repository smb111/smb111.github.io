原码反码补码

发表于 2017-03-22   |   分类于 C++

机器数和真值

1、机器数

一个数在计算机中的表现形式（注意并不是存储形式），机器数是带符号的

比如十进制的 +3 ,计算机字长是8位那么，转换成二进制数就是 : 0000 0011

如果十进制的 -3 ,二进制数是 ： 1000 0011

2、真值

0000 0001的真值 = +000 0001 = +1，1000 0001的真值 = –000 0001 = –1

原码、补码、反码

原码、补码、反码 只是计算机对机器数的一种编码方式，用来存储，在计算机中使用补码来进行编码存储（至于为什么，下面会讲）

1. 原码

原码就是符号位加上真值的绝对值, 即用第一位表示符号, 其余位表示值. 比如如果是8位二进制:

[+1]原 = 0000 0001

[-1]原 = 1000 0001

第一位是符号位. 因为第一位是符号位, 所以8位二进制数的取值范围就是:

[1111 1111 , 0111 1111]

即

[-127 , 127]

原码是人脑最容易理解和计算的表示方式.

2. 反码

反码的表示方法是:

正数的反码是其本身

负数的反码是在其原码的基础上, 符号位不变，其余各个位取反.

[+1] = [00000001]原 = [00000001]反

[-1] = [10000001]原 = [11111110]反

可见如果一个反码表示的是负数, 人脑无法直观的看出来它的数值. 通常要将其转换成原码再计算.

3. 补码

补码的表示方法是:

正数的补码就是其本身

负数的补码是在其原码的基础上, 符号位不变, 其余各位取反, 最后+1. (即在反码的基础上+1)

[+1] = [00000001]原 = [00000001]反 = [00000001]补

[-1] = [10000001]原 = [11111110]反 = [11111111]补

对于负数, 补码表示方式也是人脑无法直观看出其数值的. 通常也需要转换成原码在计算其数值.

负数原码求补码 ： 符号位不变, 其余各位取反, 最后+1

负数补码求原码 ： 符号位不变, 其余各位取反, 最后+1

为什么要使用 原码 补码 反码

原码是最直观的，但是这仅仅是对人类而言，因为我们能知道第一位是表示符号位的。

而对于计算机来说，要进行第一位符号位的判断就要添加额外的电路，非常麻烦。

因为加法运算就是加上一个负数，所以计算机只设计加法运算电路。

综上：我们可以让符号位也参与运算，这样就不用设计判断电路

比如计算 ： 1 + （-1）

使用原码 ： 0000 0001 + 1000 0001 = 1000 0010 真值（-2）这是不对的

使用反码 ： 0000 0001 + 1111 1110 = 1111 1111 结果取反 1000 0000 真值（-0）

使用反码的结果是对的，但是会有 +0 和 -0 这样零的表示就有两种，使用补码可以解决这个问题

使用补码 ： 解决了0的符号以及两个编码的问题:

1-1 = 1 + (-1) = [0000 0001]原 + [1000 0001]原 = [0000 0001]补 + [1111 1111]补 = [0000 0000]补=[0000 0000]原

这样 0000 0000 就用来表示 0 ， 而 1000 0000 则用来表示 -128 : 注意因为实际上是使用以前的-0的补码来表示-128, 所以-128并没有原码和反码表示.(对-128的补码表示[1000 0000]补算出来的原码是[0000 0000]原, 这是不正确的)

为什么使用原码或反码表示的范围为[-127, +127], 而使用补码表示的范围为[-128, 127].

使用补码 ： 解决了0的符号以及两个编码的问题:

1-1 = 1 + (-1) = [0000 0001]原 + [1000 0001]原 = [0000 0001]补 + [1111 1111]补 = [0000 0000]补=[0000 0000]原

这样 0000 0000 就用来表示 0 ， 而 1000 0000 则用来表示 -128 : 注意因为实际上是使用以前的-0的补码来表示-128, 所以-128并没有原码和反码表示.(对-128的补码表示[1000 0000]补算出来的原码是[0000 0000]原, 这是不正确的)

因为机器使用补码, 所以对于编程中常用到的32位int类型, 可以表示范围是: [-231, 231-1] 因为第一位表示的是符号位.而使用补码表示时又可以多保存一个最小值.

Ｃ语言中的 printf 的使用

#define A (1u << 31)  // 这个地方使用 u 后标指示A是一个无符号的数

#define B (1 << 31)   // 不使用 u 则默认是一个有符号的数（这里编译器认为 int 占有 4 个字节） 有符号 -2147483648 -- 2147483647   无符号  0 -- 

cout << B;   // -2147483648     1000000 00000000 00000000 00000000 就是上文说到的 -0，被表示成了负的最小值（补码的原因）

cout << A;   // 2147483648

printf("%d",B);  // -2147483648   

printf("%u",B)   // 2147483648   这里有符号数转换成了无符号进行了输出

出现这种情况的原因是  :  1000000 00000000 00000000 00000000 在有符号的情况下表示 -2147483648  而在无符号的情况下就是2147483648
无符号数和有符号数进行运算的时候会转换成 无符号数 在进行运算 ^_^

以后使用变量的时候，如果确定无符号一定使用 unsinged 或者 u
